SweepFeatures Object

Derived from: Base Object
Defined in namespace "adsk::fusion" and the header file is <Fusion/Features/SweepFeatures.h>

Description

Collection that provides access to all of the existing sweep features in a component and supports the ability to create new sweep features.

Methods

Name Description
add Creates a new sweep feature.
classType Static function that all classes support that returns the type of the class as a string. The returned string matches the string returned by the objectType property. For example if you have a reference to an object and you want to check if it's a SketchLine you can use myObject.objectType == fusion.SketchLine.classType().
createInput Creates a SweepFeatureInput object for defining a simple sweep feature with only a path and no guide rail or surface. Use properties and methods on this object to define the sweep you want to create and then use the Add method, passing in the SweepFeatureInput object.
createInputForSolid Creates a SweepFeatureInput object for defining a simple sweep feature from a B-Rep solid with a path. Use properties and methods on this object to define the sweep you want to create, and then use the Add method, passing in the SweepFeatureInput object.
item Function that returns the specified sweep feature using an index into the collection.
itemByName Function that returns the specified sweep feature using the name of the feature.

Properties

Name Description
count The number of sweep features in the collection.
isValid Indicates if this object is still valid, i.e. hasn't been deleted or some other action done to invalidate the reference.
objectType This property is supported by all objects in the API and returns a string that contains the full name (namespace::objecttype) describing the type of the object.

It's often useful to use this in combination with the classType method to see if an object is a certain type. For example: if obj.objectType == adsk.core.Point3D.classType():
