OperationStrategy Object

Derived from: Base Object
Defined in namespace "adsk::cam" and the header file is <Cam/Operations/OperationStrategy.h>

Description

The OperationStrategy contains information about a strategy such as its name, title and description.

Methods

Name Description
classType Static function that all classes support that returns the type of the class as a string. The returned string matches the string returned by the objectType property. For example if you have a reference to an object and you want to check if it's a SketchLine you can use myObject.objectType == fusion.SketchLine.classType().
createFromString Create an OperationStrategy for a given name.

Properties

Name Description

isAdditiveStrategy Gets whether given OperationStrategy is an additive strategy.
isCuttingStrategy Gets whether given OperationStrategy is a cutting strategy.
isDrillingStrategy Gets whether given OperationStrategy is a drilling strategy.
isFinishingStrategy Gets whether given OperationStrategy is a finishing strategy.
isGenerationAllowed Returns true if the strategy is allowed to be generated due to licensing and enabled preview features. Some strategies also require Active machining extension to be generated. Returns false otherwise.
isMillingStrategy Gets whether given OperationStrategy is a milling strategy.
isRotaryStrategy Gets whether given OperationStrategy is a rotary strategy.
isSupportStrategy Gets whether given OperationStrategy is an additive support strategy.
isTurningStrategy Gets whether given OperationStrategy is a turning strategy.
isValid Indicates if this object is still valid, i.e. hasn't been deleted or some other action done to invalidate the reference.
name Get the name of the strategy. This is equivalent to the Operation's strategy property. Use as strategy parameter when creating a OperationInput.
objectType This property is supported by all objects in the API and returns a string that contains the full name (namespace::objecttype) describing the type of the object.
